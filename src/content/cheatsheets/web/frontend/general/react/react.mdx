# ⚛️ React Cheatsheet Completo ⚛️

React es una librería de JavaScript declarativa, eficiente y flexible para construir interfaces de usuario. Permite componer UIs complejas a partir de pequeñas y aisladas piezas de código llamadas "componentes".

---

## 1. 🌟 Conceptos Clave

* **Componentes**: Los bloques de construcción fundamentales de una aplicación React. Piezas de UI reutilizables e independientes.
* **JSX (JavaScript XML)**: Una extensión de sintaxis para JavaScript que te permite escribir HTML dentro de JavaScript. Es compilado por Babel a llamadas a `React.createElement()`.
* **Virtual DOM**: Una representación ligera en memoria del DOM real. React lo utiliza para optimizar las actualizaciones de la UI, minimizando las manipulaciones directas del DOM real.
* **Props (Propiedades)**: Argumentos o datos que se pasan de un componente padre a un componente hijo. Son de solo lectura (inmutables).
* **Estado (State)**: Datos que un componente gestiona internamente y que pueden cambiar con el tiempo, afectando el renderizado del componente. Es mutable.
* **Flujo de Datos Unidireccional (One-Way Data Flow)**: Los datos fluyen en una sola dirección: de padres a hijos a través de las props.

## 2. 🛠️ Configuración Inicial (Crear un Proyecto React)

## 3. 🧩 Componentes (Functional Components)

Los componentes funcionales son el enfoque moderno y preferido en React.

**Sintaxis Básica:**

```jsx
// src/components/MyComponent.jsx (o .tsx para TypeScript)
import React from 'react'; // React ya no es estrictamente necesario importar en JSX para versiones recientes, pero es buena práctica

function MyComponent(props) { // Recibe props como argumento
  return (
    <div>
      <h1>Hola desde MyComponent</h1>
      <p>{props.message}</p> {/* Usando una prop */}
    </div>
  );
}

export default MyComponent;
```

**Uso en otro Componente:**

```jsx
// src/App.jsx
import React from 'react';
import MyComponent from './components/MyComponent';

function App() {
  return (
    <div>
      <MyComponent message="¡Bienvenido a React!" />
      <MyComponent message="Este es otro mensaje." />
    </div>
  );
}

export default App;
```

## 4. 📝 JSX (JavaScript XML)

### 4.1. Reglas Básicas

* **Un solo elemento raíz**: Cada componente debe devolver un solo elemento raíz (puedes usar un `Fragment` `<></>` si no quieres un `div` extra).
* **Cierre de etiquetas**: Todas las etiquetas deben cerrarse (`<img />`, `<br />`, `<div></div>`).
* **`className` en lugar de `class`**: `class` es una palabra reservada en JS, usa `className` para atributos CSS.
* **`htmlFor` en lugar de `for`**: Similar al anterior, para etiquetas `label`.
* **CamelCase para atributos HTML**: `onClick`, `tabIndex`.

```jsx
// Fragment (para evitar un div extra)
<>
  <h1>Título</h1>
  <p className="my-paragraph">Un párrafo.</p>
  <label htmlFor="myInput">Input:</label>
  <input id="myInput" onClick={() => console.log('Click!')} />
</>
```

### 4.2. Incrustar Expresiones JavaScript `{}`

Puedes incrustar cualquier expresión JavaScript válida dentro de llaves `{}`.

```jsx
function Greeting(props) {
  const name = props.name || 'Invitado';
  const age = 30;
  const isAdult = age >= 18;

  return (
    <div>
      <h2>Hola, {name.toUpperCase()}!</h2> {/* Accediendo a variable y método */}
      <p>Tu edad es: {age}</p>
      {isAdult && <p>Eres un adulto.</p>} {/* Renderizado condicional */}
      <p>La suma es: {10 + 5}</p> {/* Operación matemática */}
    </div>
  );
}
```

## 5. 🎁 Props (Propiedades)

* Se pasan de padre a hijo.
* Son inmutables (el componente hijo NO debe modificarlas).

```jsx

function ProductCard({ name, price, imageUrl }) { // Destructuring de props
  return (
    <div className="product-card">
      <img src={imageUrl} alt={name} />
      <h3>{name}</h3>
      <p>${price.toFixed(2)}</p>
      <button>Añadir al carrito</button>
    </div>
  );
}
export default ProductCard;

// Padre: App.jsx
import ProductCard from './components/ProductCard';

function App() {
  return (
    <div>
      <ProductCard
        name="Laptop Pro"

      <ProductCard
        name="Mouse Ergonómico"
        price={25.99}
        imageUrl="https://via.placeholder.com/150"
```

## 6. 🔄 Estado (State) con Hooks

Los Hooks son funciones especiales que te permiten "engancharte" a las características de React desde componentes funcionales.

### 6.1. `useState` (Estado Local)

* Permite a los componentes funcionales tener estado.
* Devuelve un par: `[currentState, setStateFunction]`.

```jsx
import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0); // Valor inicial: 0

  const increment = () => {
    setCount(prevCount => prevCount + 1); // Usar función de actualización para el estado anterior
  };

  const decrement = () => {
    setCount(count - 1); // También puedes pasar el nuevo valor directamente
  };

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={increment}>Incrementar</button>
      <button onClick={decrement}>Decrementar</button>
    </div>
  );
}
export default Counter;
```

### 6.2. `useEffect` (Efectos Secundarios)

* Se utiliza para manejar efectos secundarios (operaciones que interactúan con el mundo exterior: fetch de datos, suscripciones, manipulaciones manuales del DOM, timers).
* Se ejecuta **después de cada renderizado** del componente (por defecto).
* **Array de dependencias (`deps`):**
  * `[]` (array vacío): Se ejecuta **solo una vez** después del primer renderizado (como `componentDidMount`).
  * `[value1, value2]`: Se ejecuta cuando `value1` o `value2` cambian.
  * Sin array de dependencias: Se ejecuta después de **cada renderizado**.
* **Función de limpieza (`cleanup`):**
  * La función retornada por `useEffect` se ejecuta cuando el componente se desmonta o antes de que el efecto se re-ejecute.

```jsx
import React, { useState, useEffect } from 'react';

function DataFetcher({ userId }) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    console.log('Efecto ejecutado para userId:', userId);
    setLoading(true);
    setError(null);
    setData(null); // Limpiar datos anteriores

    const fetchData = async () => {
      try {
        const response = await fetch(`https://jsonplaceholder.typicode.com/users/${userId}`);
        if (!response.ok) {
          throw new Error(`HTTP error! status: ${response.status}`);
        }
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err);
      } finally {
        setLoading(false);
      }
    };

    fetchData();

    // Función de limpieza (se ejecuta al desmontar o antes de re-ejecutar el efecto)
    return () => {
      console.log('Limpiando efecto para userId:', userId);
      // Aquí puedes cancelar suscripciones, limpiar timers, etc.
    };
  }, [userId]); // Este efecto se re-ejecuta solo cuando userId cambia

  if (loading) return <p>Cargando datos...</p>;
  if (error) return <p>Error: {error.message}</p>;
  if (!data) return null; // No hay datos aún

  return (
    <div>
      <h3>Detalles del Usuario {data.id}</h3>
      <p>Nombre: {data.name}</p>
      <p>Email: {data.email}</p>
    </div>
  );
}
export default DataFetcher;
```

### 6.3. `useContext` (Estado Global)

* Permite compartir datos entre componentes sin pasar props manualmente a cada nivel del árbol (prop drilling).
* Proceso:
  1. Crea un contexto (`React.createContext()`).
  2. Envuelve los componentes que necesitan acceso al contexto con `Context.Provider`.
  3. Usa `useContext(Context)` en los componentes hijos para acceder al valor.

```jsx
// 1. userContext.js
import React, { createContext, useState, useContext } from 'react';

const UserContext = createContext(null); // Valor inicial

export const UserProvider = ({ children }) => {
  const [user, setUser] = useState({ name: 'Invitado', isLoggedIn: false });

  const login = (username) => setUser({ name: username, isLoggedIn: true });
  const logout = () => setUser({ name: 'Invitado', isLoggedIn: false });

  return (
    <UserContext.Provider value={{ user, login, logout }}>
      {children}
    </UserContext.Provider>
  );
};

// Hook personalizado para facilitar el uso
export const useUser = () => useContext(UserContext);

// 2. Componente que provee el contexto (ej. App.jsx)
import { UserProvider } from './contexts/UserContext';
import ProfileDisplay from './components/ProfileDisplay';
import AuthButtons from './components/AuthButtons';

function App() {
  return (
    <UserProvider> {/* Envuelve los componentes que necesitan acceso al usuario */}
      <h1>Mi Aplicación</h1>
      <ProfileDisplay />
      <AuthButtons />
    </UserProvider>
  );
}

// 3. Componente que consume el contexto (ej. ProfileDisplay.jsx)
import React from 'react';
import { useUser } from '../contexts/UserContext';

function ProfileDisplay() {
  const { user } = useUser(); // Accede al contexto
  return (
    <div>
      {user.isLoggedIn ? (
        <p>Bienvenido, {user.name}!</p>
      ) : (
        <p>Por favor, inicia sesión.</p>
      )}
    </div>
  );
}

// 4. Otro componente que consume el contexto (ej. AuthButtons.jsx)
import React from 'react';
import { useUser } from '../contexts/UserContext';

function AuthButtons() {
  const { user, login, logout } = useUser();
  return (
    <div>
      {user.isLoggedIn ? (
        <button onClick={logout}>Cerrar Sesión</button>
      ) : (
        <button onClick={() => login('Alice')}>Iniciar Sesión</button>
      )}
    </div>
  );
}
```

### 6.4. `useRef` (Referencias y Valores Mutables)

* Devuelve un objeto `ref` mutable cuya propiedad `.current` puede almacenar un valor.
* Útil para acceder a elementos del DOM o para almacenar valores mutables que no necesitan disparar un re-render.

```jsx
import React, { useRef } from 'react';

function MyForm() {
  const inputRef = useRef(null); // Crea una ref para el input

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Valor del input: ${inputRef.current.value}`);
    inputRef.current.focus(); // Enfocar el input
  };

  return (
    <form onSubmit={handleSubmit}>
      <input type="text" ref={inputRef} /> {/* Asigna la ref al input */}
      <button type="submit">Obtener Valor</button>
    </form>
  );
}
export default MyForm;
```

### 6.5. `useCallback` (Memorización de Funciones)

* Devuelve una versión memorizada de la función de callback que solo cambia si una de sus dependencias ha cambiado.
* Útil para optimizar componentes hijos que se re-renderizan innecesariamente si sus props de función cambian en cada render del padre.

```jsx
import React, { useState, useCallback } from 'react';
import ChildComponent from './ChildComponent'; // Un componente hijo que podría ser Memo

function ParentComponent() {
  const [count, setCount] = useState(0);
  const [name, setName] = useState('Alice');

  // Esta función se recrea en cada render
  // const handleClick = () => {
  //   setCount(count + 1);
  // };

  // Esta función solo se recrea si 'count' cambia
  const handleClick = useCallback(() => {
    setCount(prevCount => prevCount + 1);
  }, []); // Dependencia vacía significa que la función se crea una vez

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={handleClick}>Incrementar</button>
      <input value={name} onChange={e => setName(e.target.value)} />
      <ChildComponent onButtonClick={handleClick} /> {/* Pasa la función memorizada */}
    </div>
  );
}
export default ParentComponent;
```

### 6.6. `useMemo` (Memorización de Valores Calculados)

* Devuelve un valor memorizado que solo se recalcula cuando una de sus dependencias cambia.
* Útil para cálculos costosos.

```jsx
import React, { useState, useMemo } from 'react';

function ExpensiveCalculation() {
  const [num, setNum] = useState(0);
  const [darkMode, setDarkMode] = useState(false);

  // Esta función se ejecuta en cada render si no está memorizada
  const calculateFactorial = (n) => {
    console.log('Calculando factorial...');
    if (n <= 0) return 1;
    let result = 1;
    for (let i = 1; i <= n; i++) {
      result *= i;
    }
    return result;
  };

  // El factorial solo se recalcula cuando 'num' cambia
  const factorial = useMemo(() => calculateFactorial(num), [num]);

  return (
    <div style={{ background: darkMode ? 'black' : 'white', color: darkMode ? 'white' : 'black' }}>
      <input type="number" value={num} onChange={e => setNum(Number(e.target.value))} />
      <p>Factorial de {num} es: {factorial}</p>
      <button onClick={() => setDarkMode(!darkMode)}>Toggle Dark Mode</button>
    </div>
  );
}
export default ExpensiveCalculation;
```

### 6.7. `useReducer` (Estado Complejo)

* Una alternativa a `useState` para lógica de estado más compleja, especialmente cuando el estado es un objeto o array y las actualizaciones dependen del estado anterior o involucran múltiples sub-valores.
* Funciona con un patrón similar a Redux: `(state, action) => newState`.

```jsx
import React, { useReducer } from 'react';

// 1. Definir el estado inicial
const initialState = { count: 0 };

// 2. Definir la función reducer
function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    case 'reset':
      return initialState;
    default:
      throw new Error();
  }
}

function CounterWithReducer() {
  const [state, dispatch] = useReducer(reducer, initialState); // state es el estado actual, dispatch es la función para enviar acciones

  return (
    <div>
      <p>Contador: {state.count}</p>
      <button onClick={() => dispatch({ type: 'increment' })}>Incrementar</button>
      <button onClick={() => dispatch({ type: 'decrement' })}>Decrementar</button>
      <button onClick={() => dispatch({ type: 'reset' })}>Resetear</button>
    </div>
  );
}
export default CounterWithReducer;
```

## 7. 🚦 Renderizado Condicional

Renderiza diferentes elementos o componentes basados en una condición.

* **`if`/`else` (fuera de JSX):**

  ```jsx
  function Greeting({ isLoggedIn }) {
    if (isLoggedIn) {
      return <h1>Bienvenido de nuevo!</h1>;
    }
    return <h1>Por favor, inicia sesión.</h1>;
  }
  ```
* **Operador Ternario (`condition ? true : false`):**

  ```jsx
  function Greeting({ isLoggedIn }) {
    return (
      <h1>{isLoggedIn ? 'Bienvenido de nuevo!' : 'Por favor, inicia sesión.'}</h1>
    );
  }
  ```
* **Operador Lógico `&&` (solo si la condición es `true`):**

  ```jsx
  function AdminPanel({ isAdmin }) {
    return (
      <div>
        {isAdmin && <button>Ver estadísticas</button>}
        {!isAdmin && <p>No tienes permiso para ver esto.</p>}
      </div>
    );
  }
  ```

## 8. 📜 Renderizado de Listas

Utiliza el método `.map()` en arrays para renderizar listas de elementos. **¡Siempre usa la prop `key`!**

* **`key` Prop**: Una cadena o número único que React utiliza para identificar elementos en una lista. Ayuda a React a optimizar el renderizado y a gestionar la identidad de los elementos cuando la lista cambia.
  * Debe ser único entre los hermanos.
  * Idealmente, usa un ID estable del elemento de datos. Evita usar el índice del array si la lista puede cambiar de orden, añadir/eliminar elementos.

```jsx
function ItemList({ items }) {
  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>{item.name}</li> // item.id como key
      ))}
    </ul>
  );
}

// Ejemplo de uso
const products = [
  { id: 1, name: 'Laptop' },
  { id: 2, name: 'Teclado' },
  { id: 3, name: 'Monitor' },
];

function App() {
  return <ItemList items={products} />;
}
```

## 9. ⚡ Manejo de Eventos

* Los eventos en React se nombran usando camelCase (ej. `onClick`, `onChange`).
* Se les pasa una función como manejador de evento.
* React usa "Synthetic Events", que son envolturas alrededor de los eventos nativos del navegador para garantizar la consistencia entre navegadores.

```jsx
function MyButton() {
  const handleClick = (event) => {
    console.log('Botón clickeado!', event); // El objeto de evento sintético
  };

  const handleChange = (event) => {
    console.log('Input cambió:', event.target.value); // Acceder al valor del input
  };

  return (
    <div>
      <button onClick={handleClick}>Clickeame</button>
      <input type="text" onChange={handleChange} placeholder="Escribe algo" />
    </div>
  );
}
```

## 10. 📝 Formularios

En React, los inputs de los formularios pueden ser "controlados" o "no controlados".

### 10.1. Componentes Controlados (Recomendado)

* El valor del input es controlado por el estado de React.
* Necesitas un manejador de `onChange` para actualizar el estado cada vez que cambia el input.

```jsx
import React, { useState } from 'react';

function ControlledForm() {
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');

  const handleSubmit = (event) => {
    event.preventDefault(); // Previene el comportamiento por defecto de recarga
    alert(`Nombre: ${name}, Email: ${email}`);
    // Aquí normalmente enviarías los datos a una API
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Nombre:
        <input type="text" value={name} onChange={e => setName(e.target.value)} />
      </label>
      <br />
      <label>
        Email:
        <input type="email" value={email} onChange={e => setEmail(e.target.value)} />
      </label>
      <br />
      <button type="submit">Enviar</button>
    </form>
  );
}
export default ControlledForm;
```

### 10.2. Componentes No Controlados (Menos común)

* El DOM gestiona el estado del input. Usas una `ref` para acceder al valor cuando lo necesitas (ej. al enviar el formulario).
* Menos React-ish, pero puede ser útil para formularios muy simples o para integrar con librerías de terceros.

```jsx
import React, { useRef } from 'react';

function UncontrolledForm() {
  const nameInputRef = useRef(null);
  const emailInputRef = useRef(null);

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Nombre: ${nameInputRef.current.value}, Email: ${emailInputRef.current.value}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Nombre:
        <input type="text" ref={nameInputRef} />
      </label>
      <br />
      <label>
        Email:
        <input type="email" ref={emailInputRef} />
      </label>
      <br />
      <button type="submit">Enviar</button>
    </form>
  );
}
export default UncontrolledForm;
```

## 11. 🎨 Estilización en React

* **CSS en línea (Inline CSS)**: Objetos JavaScript con propiedades camelCase.
  ```jsx
  <h1 style={{ color: 'blue', fontSize: '24px' }}>Mi Título</h1>
  ```
* **Hojas de estilo CSS (CSS Stylesheets)**: El método más común. Importa archivos `.css`.
  ```jsx
  // MyComponent.css
  .my-button {
    background-color: green;
    color: white;
  }

  // MyComponent.jsx
  import './MyComponent.css';
  <button className="my-button">Mi Botón</button>
  ```
* **Módulos CSS (CSS Modules)**: Evita conflictos de nombres de clases al generar nombres de clases únicos. (`.module.css`).
  ```jsx
  // MyComponent.module.css
  .button {
    background-color: purple;
    color: white;
  }

  // MyComponent.jsx
  import styles from './MyComponent.module.css';
  <button className={styles.button}>Mi Botón</button>
  ```
* **Librerías CSS-in-JS**: Como Styled Components, Emotion. Permiten escribir CSS directamente en tus componentes JavaScript.
  * Ej: `npm install styled-components`

  ```jsx
  // MyStyledComponent.jsx
  import styled from 'styled-components';

  const StyledButton = styled.button`
    background-color: hotpink;
    color: white;
    padding: 10px 20px;
    border-radius: 5px;
  `;

  function MyComponent() {
    return <StyledButton>Botón Estilizado</StyledButton>;
  }
  export default MyComponent;
  ```
* **Frameworks CSS de Utilidad**: Como Tailwind CSS. Aplican clases predefinidas para estilos rápidos.

## 13. 💡 Buenas Prácticas y Consejos

* **Componentes Pequeños y Reutilizables**: Divide tu UI en componentes pequeños y específicos con una única responsabilidad.
* **Separación de Preocupaciones**: Separa la lógica de presentación de la lógica de negocio.
* **Prop Drilling vs. Context API**: Evita pasar props por muchos niveles (prop drilling). Usa Context API o librerías de gestión de estado (Redux, Zustand) para estado global.
* **Inmutabilidad del Estado**: Cuando actualices el estado (especialmente objetos o arrays), crea nuevas copias en lugar de mutar el estado directamente.
  ```javascript
  // Incorrecto:
  // const newArr = myArray; newArr.push(item); setMyArray(newArr);
  // Correcto:
  setMyArray(prevArray => [...prevArray, item]);
  setMyObject(prevObject => ({ ...prevObject, newProp: 'value' }));
  ```
* **Keys Adecuadas**: Asegúrate de que las `key`s de tus listas sean únicas y estables.
* **Manejo de Formularios Controlados**: Prefiere componentes controlados para formularios para tener un mejor control sobre el estado del input.
* **Optimización con `React.memo`**: Envuelve componentes funcionales con `React.memo` para que no se re-rendericen si sus props no han cambiado. Útil junto con `useCallback` y `useMemo`.
* **Manejo de Errores (Error Boundaries)**: Utiliza `Error Boundaries` (solo componentes de clase) para capturar y manejar errores en el árbol de componentes sin romper toda la aplicación.
* **Tipado (TypeScript)**: Usa TypeScript para añadir seguridad de tipos a tus componentes, props, estado y más.
* **ESLint/Prettier**: Configura estas herramientas para mantener un código consistente y limpio.
