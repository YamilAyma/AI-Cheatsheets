# 📦 Amazon S3 (Simple Storage Service) Cheatsheet Completo 📦

**Amazon S3** es un servicio de almacenamiento de objetos escalable, duradero, seguro y de alta disponibilidad. Permite almacenar y recuperar cualquier cantidad de datos desde cualquier lugar de la web. Es la columna vertebral de muchas arquitecturas en la nube.

---

## 1. 🌟 Conceptos Clave

* **Bucket (Cubo)**: Un contenedor lógico para almacenar objetos en S3. Es el nivel superior de organización.
  * **Nombres de Bucket**: Deben ser globalmente únicos en S3 (en todas las regiones y cuentas), siguiendo convenciones DNS (sin mayúsculas, guiones bajos, etc.).
  * **Región de Bucket**: Cada bucket se crea en una región específica de AWS.
* **Objeto (Object)**: El elemento fundamental almacenado en S3. Es un archivo (cualquier tipo de archivo) más sus metadatos.
  * **Clave de Objeto (Object Key)**: El nombre único del objeto dentro de un bucket (incluye el "path" virtual).
  * **Metadatos**: Información sobre el objeto (ej. `Content-Type`, `Date`, `Size`, metadatos personalizados).
  * **Version ID**: Si el versionamiento está habilitado, cada versión de un objeto tiene un ID único.
* **Durabilidad**: S3 está diseñado para una durabilidad del 99.999999999% (once nines) de los objetos, distribuyendo los datos en múltiples dispositivos y a través de múltiples Zonas de Disponibilidad.
* **Disponibilidad**: S3 ofrece alta disponibilidad (99.99% para S3 Standard).
* **Escalabilidad Ilimitada**: La capacidad de almacenamiento es prácticamente ilimitada.
* **Pago por Uso**: Solo pagas por el almacenamiento que usas, la transferencia de datos y las solicitudes.
* **Eventos S3**: Disparan notificaciones (a Lambda, SQS, SNS) cuando ocurren eventos en un bucket (ej. `ObjectCreated`, `ObjectDeleted`).

## 2. 🛠️ Acceso y Gestión

### 2.1. AWS Management Console (UI)

* Interfaz web para crear buckets, subir objetos, configurar permisos, etc.

### 2.2. AWS CLI (Command Line Interface)

* Herramienta de línea de comandos. Requiere `aws configure`.
  * **`aws s3 ls`**: Lista buckets.
  * **`aws s3 mb s3://my-new-bucket`**: Crea un bucket (`mb` = make bucket).
  * **`aws s3 rb s3://my-new-bucket`**: Elimina un bucket (`rb` = remove bucket).
  * **`aws s3 cp my_local_file.txt s3://my-bucket/path/to/remote_file.txt`**: Copia un archivo local a S3.
  * **`aws s3 cp s3://my-bucket/remote_file.txt my_local_file.txt`**: Copia de S3 a local.
  * **`aws s3 sync . s3://my-bucket/docs`**: Sincroniza un directorio local con un prefijo S3.
  * **`aws s3 rm s3://my-bucket/object.txt`**: Elimina un objeto.
  * **`aws s3 rm s3://my-bucket --recursive`**: Elimina todos los objetos de un bucket (para eliminar el bucket).

### 2.3. AWS SDKs (Software Development Kits)

* Bibliotecas para programar con S3 en varios lenguajes (Python Boto3, Java SDK, Node.js SDK, .NET SDK).
  ```python
  # Python (Boto3)
  import boto3

  s3 = boto3.client('s3', region_name='us-east-1')

  # Subir un archivo
  s3.upload_file('local_file.txt', 'my-bucket-123', 'remote_folder/remote_file.txt')

  # Descargar un archivo
  s3.download_file('my-bucket-123', 'remote_folder/remote_file.txt', 'downloaded_file.txt')

  # Listar objetos
  response = s3.list_objects_v2(Bucket='my-bucket-123', Prefix='remote_folder/')
  for obj in response.get('Contents', []):
      print(obj['Key'])
  ```

## 3. 💾 Clases de Almacenamiento (Storage Classes)

Diferentes opciones optimizadas para costos y patrones de acceso.

* **S3 Standard**:
  * **Uso**: Datos de uso frecuente, rendimiento de baja latencia.
  * **Disponibilidad**: 99.99%.
  * **Durabilidad**: 99.999999999%.
  * **Costo**: Mayor.
* **S3 Intelligent-Tiering**:
  * **Uso**: Datos con patrones de acceso desconocidos o cambiantes. Mueve automáticamente los datos entre tiers de acceso frecuente y poco frecuente.
  * **Costo**: Variable, optimiza automáticamente.
* **S3 Standard-IA (Infrequent Access)**:
  * **Uso**: Datos a los que se accede con poca frecuencia, pero requieren acceso rápido cuando se necesitan.
  * **Costo**: Más bajo que Standard, con un pequeño cargo por recuperación.
* **S3 One Zone-IA**:
  * **Uso**: Datos de acceso poco frecuente, pero que no requieren el mismo nivel de resiliencia (almacenado en una sola AZ).
  * **Costo**: Más bajo que Standard-IA.
* **S3 Glacier Instant Retrieval**:
  * **Uso**: Acceso instantáneo a archivos archivados (milisegundos).
  * **Costo**: Archivo muy bajo, cargo por recuperación moderado.
* **S3 Glacier Flexible Retrieval**:
  * **Uso**: Datos archivados a largo plazo (horas a minutos de recuperación).
  * **Costo**: Muy bajo, cargo por recuperación variable.
* **S3 Glacier Deep Archive**:
  * **Uso**: Archivo de datos a muy largo plazo (12 horas de recuperación).
  * **Costo**: Extremadamente bajo.

## 4. 🔒 Seguridad

### 4.1. Políticas de Bucket (Bucket Policies)

* Documentos JSON que definen permisos para un bucket y sus objetos.
* Permiten o deniegan acciones para principios específicos (usuarios, roles, cuentas) y recursos (buckets, objetos).
* Se aplican a **todos** los objetos dentro de un bucket.

  ```json
  {
    "Version": "2012-10-17",
    "Statement": [
      {
        "Sid": "PublicReadGetObject",
        "Effect": "Allow",
        "Principal": "*", // Acceso público (¡CUIDADO!)
        "Action": [ "s3:GetObject" ],
        "Resource": [ "arn:aws:s3:::my-public-bucket/*" ] // Acceso a todos los objetos
      }
    ]
  }
  ```

### 4.2. Listas de Control de Acceso (ACLs - Access Control Lists)

* Un mecanismo heredado para gestionar permisos de bucket y objeto.
* **¡Las políticas de bucket son generalmente preferidas y más potentes!**
* ACLs se usan para conceder permisos a otras cuentas de AWS o a grupos predefinidos de S3.

### 4.3. Roles de IAM (IAM Roles)

* **Para acceso desde servicios de AWS**: Es la forma **recomendada** para que las instancias EC2, funciones Lambda o otros servicios de AWS accedan a S3.
* Asocia una Política de IAM a un rol, y el servicio asume ese rol.

### 4.4. Cifrado (Encryption)

* **Cifrado en Reposo (Encryption at Rest)**: Cifra los datos mientras están almacenados en S3.
  * **SSE-S3**: S3 gestiona las claves (claves gestionadas por AWS S3).
  * **SSE-KMS**: AWS Key Management Service gestiona las claves.
  * **SSE-C**: El cliente proporciona sus propias claves.
  * **CSE (Client-Side Encryption)**: El cliente cifra los datos antes de subirlos a S3.
* **Cifrado en Tránsito (Encryption in Transit)**: Usa HTTPS/SSL. S3 soporta HTTPS por defecto.

## 5. 🧰 Características Adicionales

### 5.1. Versionamiento (Versioning)

* Cuando está habilitado en un bucket, S3 mantiene múltiples versiones de un objeto cuando se modifica o elimina.
* Protege contra eliminaciones accidentales y permite revertir a versiones anteriores.
* Aumenta el almacenamiento y, por lo tanto, el costo.

### 5.2. Replicación (Replication)

* **Replicación entre Regiones (Cross-Region Replication - CRR)**: Replica objetos a un bucket en una región diferente. Para recuperación ante desastres y menor latencia para usuarios en diferentes ubicaciones.
* **Replicación en la Misma Región (Same-Region Replication - SRR)**: Replica objetos a un bucket diferente en la misma región. Para agregación de logs, procesamiento en vivo.

### 5.3. Control de Acceso a Sitios Web Estáticos

* Puedes configurar un bucket S3 para alojar un sitio web estático.
* `index.html` (documento de índice), `error.html` (documento de error).
* Se accede a través de un endpoint de sitio web S3 (ej. `http://my-bucket.s3-website-us-east-1.amazonaws.com`).
* **¡No soporta lógica del lado del servidor (PHP, Node.js)!**
* A menudo se usa con CloudFront (CDN) para HTTPS y dominios personalizados.

### 5.4. Hosting de Objetos Públicos (S3 Public Access)

* Por defecto, todos los buckets son privados.
* Puedes hacer objetos públicos a través de:
  * Políticas de bucket.
  * ACLs.
  * **Acceso Público a Bloques (Block Public Access - BPA)**: Una configuración de seguridad a nivel de cuenta/bucket que sobrescribe los permisos de acceso público para evitar exposiciones accidentales. Por defecto, Bloquea todo acceso público.
* **URLs Pre-firmadas (Pre-signed URLs)**:
  * Otorga acceso temporal y limitado a objetos privados a usuarios sin credenciales de AWS.
  * Generado con un SDK de AWS.

### 5.5. Transfer Acceleration

* Mejora las transferencias de archivos a S3 usando los Edge Locations de CloudFront. Útil para usuarios geográficamente dispersos o archivos muy grandes.

### 5.6. S3 Object Lock

* Protege objetos de ser eliminados o sobrescritos durante un período de tiempo fijo o indefinido (cumplimiento normativo).

### 5.7. Gestión del Ciclo de Vida (Lifecycle Management)

* Automatiza el movimiento de objetos entre clases de almacenamiento o su eliminación después de un tiempo.
* **Ejemplo**: Mover objetos a S3 Standard-IA después de 30 días, y luego a Glacier después de 90 días, y eliminarlos después de 365 días.

## 6. 💡 Buenas Prácticas y Consejos

* **Bloquear Acceso Público por Defecto**: **Siempre mantén `Block Public Access` habilitado** en tus buckets a menos que tengas un caso de uso muy específico y justificado (ej. hosting de sitios web estáticos).
* **Principio de Mínimo Privilegio (IAM)**: Otorga solo los permisos de IAM necesarios para acceder a tus buckets y objetos S3.
* **Cifrado Activo**: Siempre cifra tus datos en S3 (SSE-S3 es gratuito y fácil).
* **Versionamiento**: Habilita el versionamiento en buckets críticos para protección contra eliminaciones/sobrescrituras accidentales.
* **Gestión del Ciclo de Vida**: Utilízala para optimizar los costos, moviendo automáticamente los datos entre clases de almacenamiento según los patrones de acceso.
* **Nombres de Bucket Únicos y Descriptivos**: Usa nombres que reflejen el propósito del bucket y que sean únicos.
* **Consistencia de Nombres de Objeto (Keys)**: Diseña una estructura de nombres de objetos (keys) lógica y jerárquica para facilitar la organización y las consultas (ej. `folder/subfolder/file.txt`).
* **S3 Events**: Aprovecha los eventos S3 para disparar lambdas (ej. para procesar imágenes, datos) cuando se suben nuevos objetos.
* **Monitoreo (CloudWatch)**: Monitoriza el uso de S3 (solicitudes, errores, uso de almacenamiento) con CloudWatch.
* **CORS Configuration**: Si alojas activos estáticos en S3 y tu aplicación se ejecuta en otro dominio, configura las reglas CORS en el bucket.
* **Con CloudFront para Sitios Estáticos**: Para sitios web estáticos alojados en S3, usa CloudFront para HTTPS, dominio personalizado y mejor rendimiento global.
